 System Overview

**What is the main function of your system?**

CignalCare is a web-based service management and customer support system designed to streamline cable service operations, including customer account management, ticketing and complaint handling, troubleshooting assistance, technician request processing, and prepaid load or subscription management. 
The system provides a centralized platform for both customers and administrators to efficiently manage service concerns, reduce response time, and improve overall service delivery. It also includes a structured workflow for ticket resolution, technician scheduling, and prepaid transaction processing, ensuring organized and transparent operations.

Who are the users?

Administrator - Manages customers, service plans, tickets, troubleshooting guides, prepaid transactions, technician requests, and system analytics.

Subscriber / Customer - Submits service concerns, accesses troubleshooting guides, requests technician assistance, manages account inquiries, and views prepaid/load transactions.

Technician (Service Personnel) - Receives assigned service requests, updates job status, and performs on-site troubleshooting and installation support 


 System Modules / Features

-  User Authentication and Account Verification (CCA-based login system)
-  Ticketing System (Customer complaints and service concerns)
-  Troubleshooting Assistance Module (guided self-service support)
-  Technician Request and Scheduling System
-  Prepaid Load and Subscription Management
-  POS / Load Processing System (Admin-side)
-  Transaction History and Financial Monitoring
-  Customer Account Management
-  Messaging and Support Communication System
-  Admin Dashboard with Analytics and Reports
-  Remote Load Request System (GCash / Maya simulation or processing flow)

 System Architecture

CignalCare follows a full-stack client-server architecture:

- Frontend handles user interface and interactions
- Backend manages business logic and API requests
- Database stores customer, ticket, transaction, and service data
